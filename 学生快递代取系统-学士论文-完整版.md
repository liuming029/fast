# 学生快递代取系统的设计与实现

---

## 摘要

随着电子商务的快速发展，快递业务量呈现爆发式增长。高校作为人员密集、网购频繁的场所，快递包裹数量庞大，学生取件面临时间冲突、距离较远等问题。为解决这一痛点，本文设计并实现了一套基于微信小程序的学生快递代取系统。

系统采用前后端分离架构，前端使用微信小程序原生开发，后端基于 Spring Boot 框架，使用 MyBatis 进行数据持久化，MySQL 作为数据库存储。系统主要功能包括：用户注册登录、快递下单、骑手接单、订单管理、通知公告、用户管理等。通过 JWT 实现身份认证，采用 RESTful API 进行前后端数据交互。

测试结果表明，系统运行稳定，功能完善，能够有效解决高校学生取件难的问题，具有良好的实用价值和推广前景。

**关键词**：快递代取；微信小程序；Spring Boot；MyBatis；前后端分离

---

## Abstract

With the rapid development of e-commerce, the volume of express delivery business has shown explosive growth. As a place with dense population and frequent online shopping, colleges and universities have a large number of express packages, and students face problems such as time conflicts and long distances when picking up packages. To solve this pain point, this paper designs and implements a student express pickup system based on WeChat Mini Program.

The system adopts a front-end and back-end separation architecture. The front-end is developed using WeChat Mini Program natively, the back-end is based on the Spring Boot framework, uses MyBatis for data persistence, and MySQL as database storage. The main functions of the system include: user registration and login, express order placement, rider order taking, order management, notice announcements, user management, etc. Identity authentication is achieved through JWT, and RESTful API is used for front-end and back-end data interaction.

Test results show that the system runs stably and has complete functions, which can effectively solve the problem of difficulty for college students to pick up packages, and has good practical value and promotion prospects.

**Keywords**: Express pickup; WeChat Mini Program; Spring Boot; MyBatis; Front-end and back-end separation

## 1 引言

### 1.1 研究背景

近年来，随着移动互联网技术的快速发展和智能手机的普及，电子商务行业迎来了前所未有的发展机遇。根据国家邮政局发布的数据显示，2023年我国快递业务量达到1320亿件，同比增长18.6%，连续多年位居世界第一。

高校作为年轻人聚集的场所，大学生是网购的主力军。由于课程安排、实习、外出等原因，学生经常无法在快递驿站工作时间内取件，导致包裹积压、逾期退回等问题。同时，部分学生愿意利用课余时间提供代取服务赚取零花钱，但缺乏有效的信息对接平台。

传统的代取方式主要通过QQ群、微信群等社交平台发布信息，存在信息分散、沟通成本高、信任缺失、交易无保障等问题。因此，开发一个专门的学生快递代取系统具有重要的现实意义。

### 1.2 研究意义

#### 1.2.1 理论意义

本文将微信小程序、Spring Boot、MyBatis等现代Web开发技术应用于校园服务场景，探索了前后端分离架构在轻量级应用中的实践，为同类系统的开发提供了技术参考和借鉴。

#### 1.2.2 实际意义

- **方便学生取件**：解决学生因时间冲突无法取件的问题，提高取件效率。
- **提供勤工俭学机会**：为有闲暇时间的学生提供代取服务的平台，增加收入来源。
- **促进资源优化配置**：通过平台对接供需双方，提高资源利用效率。
- **推动校园信息化建设**：丰富校园服务应用，提升校园数字化水平。

### 1.3 国内外研究现状

#### 1.3.1 国外研究现状

在欧美国家，校园快递服务相对成熟。美国的Amazon、UPS等企业在高校提供专门的快递服务点，部分高校也有自己的物流管理系统。在移动应用方面，Postmates、DoorDash等平台提供同城配送服务，也可用于快递代取，但这些平台主要面向商业服务，针对校园场景的定制化功能较少。

#### 1.3.2 国内研究现状

国内在校园快递代取领域的研究和实践较为丰富。随着微信小程序的普及，出现了许多基于小程序的代取系统。学者们也对相关技术进行了研究：

马新等[1]设计了大学校园微信快递代取平台，通过用户自助下单提供需求、代取人员提供服务的模式，有效解决学生取件距离远、排队时间长的问题，同时避免了原有校内出现的代取人员来源混乱。

孙秀娟等[2]采用JSP、JS等相关技术设计基于微信小程序的校园快递代取平台，系统具有用户注册验证、下发订单、查看订单、撤销订单、意见反馈等功能，为用户提供代取和派送快递的渠道。

然而，现有系统仍存在一些不足：部分系统功能过于复杂，使用门槛高；部分系统缺乏完善的信任机制和交易保障；部分系统的骑手分成、平台收益等商业模式不够清晰。

### 1.4 研究内容与目标

#### 1.4.1 研究内容

## 3 系统设计

### 3.1 系统总体架构设计

本系统采用**前后端分离**的架构设计，将前端界面与后端业务逻辑解耦，提高了系统的可维护性和可扩展性。前后端分离架构的核心优势在于前端和后端可以独立开发、独立部署，提高了开发效率，同时便于后续的功能扩展和技术升级[4]。

#### 3.1.1 架构层次

系统分为以下几层：

1. **表现层**：微信小程序，负责页面展示和用户交互。采用微信小程序原生开发，使用WXML编写页面结构、WXSS编写页面样式、JavaScript编写业务逻辑，无需用户下载安装，即开即用[1]。

2. **网关层**：Nginx（可选），负责反向代理和负载均衡。在生产环境中，可以使用Nginx作为反向代理服务器，将请求分发到后端应用服务器，提高系统的并发处理能力。

3. **应用层**：Spring Boot应用，负责业务逻辑处理。采用Spring Boot框架，整合MyBatis持久层框架，数据库选择高性能的MySQL，采用前后端分离模式设计，通过RESTful API进行前后端数据交互[5]。

4. **数据层**：MySQL数据库，负责数据持久化存储。使用MySQL 8.x关系型数据库，通过MyBatis实现数据库读写操作，确保数据的一致性和安全性[6]。

### 3.5 安全设计

#### 3.5.1 身份认证

系统采用 JWT（JSON Web Token）进行身份认证。JWT是一种开放标准（RFC 7519），用于在网络应用环境中安全地传输信息。JWT由三部分组成：Header（头部）、Payload（载荷）和Signature（签名），使用点号（.）连接[6]。

身份认证流程如下：

1. 用户登录成功后，后端根据用户信息生成 JWT Token，Payload中包含用户ID、用户名、角色等信息，使用密钥进行签名。
2. 前端将 Token 存储在微信小程序的本地存储（wx.setStorageSync）中。
3. 后续每次请求时，前端在 HTTP Header 的 Authorization 字段中携带 Token，格式为 "Bearer {token}"。
4. 后端的 JwtAuthenticationTokenFilter 拦截所有请求，验证 Token 的有效性：检查签名是否正确、Token是否过期等。
5. Token 有效期设置为 300 分钟，过期后需要重新登录，确保系统安全性。

#### 3.5.2 权限控制

系统基于角色进行权限控制（RBAC，Role-Based Access Control）。RBAC是一种广泛使用的权限控制模型，通过为用户分配角色，再为角色分配权限，实现灵活的权限管理[5]。

系统定义了三种角色：

- **普通用户**：可以下单、查看自己的订单、查看通知、管理个人信息、申请成为骑手等。
- **骑手**：在普通用户权限基础上，可以接单、查看接单记录、管理收益、查看配送中的订单等。
- **管理员**：拥有所有权限，可以管理用户、骑手、订单、基础数据（快递站点、宿舍楼、包裹规格）、发布通知、查看数据统计等。

在后端实现中，通过拦截器或AOP（面向切面编程）对用户角色进行验证，确保用户只能访问其权限范围内的资源。

#### 3.5.3 数据安全

系统从多个层面保障数据安全：

1. **密码存储**：用户密码在数据库中加密存储。虽然当前系统使用简单的密码比较，但在实际生产环境中，建议使用BCrypt等强加密算法对密码进行哈希处理，BCrypt算法自带盐值，能够有效防止彩虹表攻击[5]。

2. **数据传输**：推荐使用 HTTPS 协议进行数据传输。HTTPS在HTTP的基础上加入了SSL/TLS协议，能够对传输的数据进行加密，防止数据被窃取或篡改，确保数据在传输过程中的机密性和完整性[6]。

3. **SQL注入防护**：使用 MyBatis 的参数化查询，防止 SQL 注入攻击。MyBatis使用 #{} 语法进行参数绑定，会自动将参数转换为预编译语句，有效防止恶意SQL代码的注入[6]。

4. **输入验证**：对用户输入的数据进行严格的验证和过滤，防止XSS（跨站脚本攻击）和CSRF（跨站请求伪造）等攻击。

```java
@RestController
public class SysLoginController {
    @Resource
    private IUserService userService;
    
    @Resource
    private TokenService tokenService;

    @PostMapping("/login")
    public AjaxResult login(@RequestBody LoginBody loginBody) {
        // 验证用户名和密码
        User user = userService.selectUserByUserName(loginBody.getUsername());
        if (user == null) {
            throw new RuntimeException("用户名或密码错误");
        }

        // 密码比较
        if (!loginBody.getPassword().equals(user.getPassword())) {
            throw new RuntimeException("密码错误");
        }

        // 构造LoginUser对象
        LoginUser loginUser = new LoginUser(user.getUserId(), user, null);

        // 生成令牌
        String token = tokenService.createToken(loginUser);
        return AjaxResult.success().put("token", token);
    }
}
```

#### 4.3.2 请求封装

**utils/request.js**：

```javascript
const baseUrl = 'http://c8c03e8.r30.cpolar.top';

function request(options) {
  return new Promise((resolve, reject) => {
    const token = wx.getStorageSync('token');
    
    wx.request({
      url: baseUrl + options.url,
      method: options.method || 'GET',
      data: options.data || {},
      header: {
        'Content-Type': 'application/json',
        'Authorization': token ? `Bearer ${token}` : ''
      },
      success: (res) => {
        if (res.statusCode === 200) {
          resolve(res.data);
        } else {
          reject(res.data);
        }
      },
      fail: (err) => {
        wx.showToast({
          title: '网络请求失败',
          icon: 'none'
        });
        reject(err);
      }
    });
  });
}
```

##### 4.3.3.2 JS 逻辑实现

**pages/login/login.js**：

```javascript
const request = require('../../utils/request.js');

Page({
  data: {
    username: '',
    password: ''
  },

  onUsernameInput(e) {
    this.setData({ username: e.detail.value });
  },

  onPasswordInput(e) {
    this.setData({ password: e.detail.value });
  },

  async login() {
    if (!this.data.username || !this.data.password) {
      wx.showToast({
        title: '请输入用户名和密码',
        icon: 'none'
      });
      return;
    }

    try {
      const res = await request({
        url: '/login',
        method: 'POST',
        data: {
          username: this.data.username,
          password: this.data.password
        }
      });

      wx.setStorageSync('token', res.token);
      wx.showToast({
        title: '登录成功',
        icon: 'success'
      });

      setTimeout(() => {
        wx.switchTab({
          url: '/pages/index/index'
        });
      }, 1500);
    } catch (err) {
      wx.showToast({
        title: err.msg || '登录失败',
        icon: 'none'
      });
    }
  },

  goToRegister() {
    wx.navigateTo({
      url: '/pages/register/register'
    });
  },

  goToForgotPassword() {
    wx.navigateTo({
      url: '/pages/forgot-password/forgot-password'
    });
  }
});
```

## 6 总结与展望

### 6.1 总结

本文设计并实现了一套基于微信小程序的学生快递代取系统，主要完成了以下工作：

1. **需求分析**：通过调研高校学生的实际需求，明确了系统的功能需求和非功能需求。
2. **系统设计**：采用前后端分离架构，设计了系统的整体架构、数据库结构、功能模块和接口规范。
3. **系统实现**：前端使用微信小程序原生开发，后端基于 Spring Boot + MyBatis 框架实现，使用 MySQL 作为数据库。
4. **系统测试**：对系统进行了功能测试、性能测试和兼容性测试，验证了系统的稳定性和可用性。

系统实现了用户注册登录、快递下单、骑手接单、订单管理、通知公告、用户管理等核心功能，运行稳定，界面友好，操作简便，能够有效解决高校学生取件难的问题。

### 6.2 不足

尽管系统已经基本实现了预期功能，但仍存在一些不足之处：

1. **支付功能缺失**：系统目前不支持在线支付，用户需要线下交易或提前充值。
2. **评价功能缺失**：缺少用户对骑手的评价功能，无法建立完善的信用体系。
3. **消息推送缺失**：缺少微信消息推送功能，订单状态变更时无法及时通知用户。
4. **数据分析不足**：缺少数据可视化和深入的数据分析功能。
5. **安全加固不足**：用户密码未使用强加密算法，建议使用 BCrypt 等加密算法。

### 6.3 展望

未来可以从以下几个方面对系统进行优化和扩展：

1. **增加在线支付功能**：接入微信支付，实现在线支付订单费用。
2. **增加评价系统**：用户可以对骑手的服务进行评价和打分，建立信用体系。
3. **增加消息推送**：使用微信小程序的订阅消息功能，及时推送订单状态变更通知。
4. **优化数据分析**：增加数据可视化图表，提供更深入的数据分析功能。
5. **增加智能调度**：基于位置信息和骑手负载，实现订单的智能分配。
6. **增加物流追踪**：对接快递物流API，实现包裹物流信息的实时追踪。
7. **优化安全机制**：使用 BCrypt 等强加密算法存储密码，增加验证码防刷机制。

随着功能的不断完善，系统将能够更好地服务于高校师生，为校园生活提供更多便利。
